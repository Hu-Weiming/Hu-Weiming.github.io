# 大模型入门：到底什么是 LLM

## Transformer 到底是什么

最近到处都在聊大模型、AI，感觉不了解点 LLM 都不好意思说自己学计算机的。但说实话，一开始我也是一头雾水。GPT、Transformer、Token 这些词到处飞，搜了好多资料才慢慢理清楚。

LLM，Large Language Model，大语言模型。说白了就是一个超大的神经网络，通过海量文本训练出来，学会了"语言"这件事。你给它一段文字，它能接着往下写——本质上就是在做下一个 Token 的预测。

底层架构是 Transformer。2017 年 Google 那篇"Attention is All You Need"提出的。

![Transformer 架构](https://upload.wikimedia.org/wikipedia/commons/thumb/8/8f/The-Transformer-model-architecture.png/440px-The-Transformer-model-architecture.png)

Transformer 的核心是注意力机制（Attention）。不用搞懂数学细节，这么理解就行：模型处理一个词的时候，会"看"句子里的其他词，判断谁跟它关系更密切。比如"我去银行取钱"，处理"银行"时模型注意到"取钱"，知道这是金融机构不是河岸。

之前的 RNN/LSTM 按顺序一个词一个词处理，长文本很慢。Transformer 可以并行处理，效率高太多了，这也是大模型能做到几百亿参数的关键。

## 预训练和微调

大模型训练分两阶段。

**预训练**：拿海量文本（网页、书籍、代码、论文），让模型预测下一个词。这么简单的目标，在足够多的数据和足够大的模型上，涌现出了各种能力。

**微调**：预训练完的模型像"什么都懂但不听话"的人。微调让它更符合人类期望。常见的是 RLHF（基于人类反馈的强化学习）——人类给回答打分，模型根据反馈调整。

ChatGPT 好用，很大功劳在微调。没微调的 base model 你问它问题它不会好好回答，只会继续"接龙"。

## Token 和上下文窗口

大模型不按"字"处理文本，而是按 Token。一个 Token 大概对应一个常见的词或子词。英文一个单词约 1-2 个 Token，中文一个字大概 1-2 个 Token。

上下文窗口（Context Window）就是模型一次能处理的最大 Token 数。GPT-4 是 128K，Claude 支持到 200K。

上下文窗口很重要——模型只能"看到"窗口内的内容。聊天聊久了，早期的对话超出窗口就被"忘掉"了。

Token 数也关系到成本。API 按 Token 收费，input 和 output 分开计价。Prompt 别废话太多，省钱。

## 常见模型对比

现在主流大模型简单聊聊：

**GPT 系列**（OpenAI）：GPT-4o 综合能力很强，API 收费，国内直接访问不了。

**Claude 系列**（Anthropic）：Claude 3.5 Sonnet / Claude 4，长文本处理强，写代码也好。上下文窗口大。

**Llama 系列**（Meta）：开源，Llama 3 效果很不错。可以本地部署，适合研究和二次开发。

**Qwen 系列**（阿里）：国产开源模型里比较能打的。Qwen2.5 各种尺寸都有，小模型本地也能跑。

我个人感受：日常用 GPT 或 Claude 都好。想折腾本地部署就用 Llama 或 Qwen。选模型看场景和预算。

## 大模型能干什么不能干什么

能干的事太多了：写代码、翻译、总结、问答、头脑风暴、文案写作等等。我现在写作业都会先让大模型给我解释一下概念，比直接搜 CSDN 效率高多了。

但也有明显的短板：

- **会胡说八道**（幻觉问题）：模型不是"知道"答案，而是在"生成"看起来合理的文本。它可能编造不存在的论文、错误的代码。
- **知识有截止日期**：训练数据有时间限制，最近的事它不知道。
- **不擅长精确计算**：让它做数学题或者逻辑推理，复杂一点的就容易出错。
- **没有真正的"理解"**：它是统计模式匹配，不是真的懂。虽然效果看起来像懂，但跟人类的理解不是一回事。

话说回来，现在大模型发展太快了。上面说的这些短板，说不定过一阵子就解决了。作为 CS 学生，我觉得了解大模型是必须的，不管以后做不做 AI 方向。
