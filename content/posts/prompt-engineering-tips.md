# Prompt Engineering 实用技巧，我总结了这些

用大模型这一年，写了不少 Prompt。从最开始的"帮我写个 xxx"，到现在能让模型按我想要的格式、风格、逻辑来输出，走了不少弯路。总结一下我觉得真正有用的技巧。

## 零样本和少样本提示

**零样本（Zero-shot）**就是直接给指令，不给例子：

```
请将以下英文翻译成中文：Hello, how are you?
```

简单任务零样本就够了。但复杂点的任务，模型可能理解不了你想要什么格式或风格。

**少样本（Few-shot）**就是给几个示例：

```
请对以下评论进行情感分类。

评论：这家店太好吃了，下次还来！
分类：正面

评论：等了一个小时才上菜，再也不来了
分类：负面

评论：东西还行，但价格偏贵
分类：
```

给了示例之后，模型就知道你要什么格式了，输出会稳定很多。

我的经验是：能用少样本就用少样本。虽然多占一些 Token，但输出质量的提升很明显。特别是你需要特定格式的时候。

## 角色设定

在 System Prompt 里给模型一个角色，效果会好不少：

```
你是一个有10年经验的 Java 后端工程师，擅长 Spring Boot 和微服务架构。
请用简洁专业的语言回答问题。
```

比起不设角色，回答会更专业、更有针对性。

我写代码的时候经常用这种角色设定："你是一个代码审查专家，请审查以下代码，指出潜在的 bug 和可以优化的地方。"效果比直接说"看看这段代码有没有问题"好很多。
